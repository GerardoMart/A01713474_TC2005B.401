Preguntas a responder:

¿Cuál es la diferencia entre Internet y la World Wide Web?

  • Básicamente, que el internet es la red, mientras que la world wide web es un servicio que usa esa red

¿Cuáles son las partes de un URL?

  • Se separan principalmente en:

    Protocolo = https

    Dominio= www.google.com

    Ruta (también llamado path) = /carpeta/pagina.html

¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

  • GET = Obtener información del servidor.

    HEAD = Es igual que GET, pero este sólo devuelve encabezados y no el cuerpo.

    POST = Enviar datos al servidor (osea crear recursos).

    PUT = Reemplazar por completo un recurso.

    PATCH = Modificar parcialmente un recurso.

    DELETE = Eliminar un recurso.

¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

  • Se debe usar el método POST porque:

    Los datos se envían en el cuerpo de la petición.

    No aparecen en el URL.

    Es más seguro que GET (aunque pues siempre debe usarse con HTTPS).

¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?

  • Se utiliza GET porque el navegador está solicitando en este caso obtener un recurso.

Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

  • Significa “OK”, por lo que la petición fue exitosa y no ocurrió ningún error.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

  • Sí.

    El error 404 significa “Not Found” (recurso no encontrado).

    Esto puede deberse a que el enlace roto,la página esté eliminada o que haya una ruta incorrecta.

    El desarrollador tiene que corregir enlaces o redirecciones.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

  • Sí.

    El error 500 significa “Internal Server Error”.

    Es un error del servidor, normalmente causado por:

    Problemas en el código backend, alguna configuración incorrecta o por fallos en la base de datos.

    Esto es completamente responsabilidad del desarrollador o administrador del servidor.

Sobre el ciclo de vida y desarrollo de los sistemas de información:

¿Cuál es el ciclo de vida de los sistemas de información?

  • Planeación = identificación de necesidades y objetivos del sistema

    Análisis = recopilación y estudio de requerimientos

    Diseño = definición de arquitectura, base de datos e interfaces

    Desarrollo = programación y construcción del sistema

    Pruebas = verificación y validación del funcionamiento

    Implementación = puesta en marcha del sistema

    Operación = uso del sistema en el entorno real

    Mantenimiento = corrección de errores y mejoras continuas

¿Cuál es el ciclo de desarrollo de sistemas de información?

  • Recolección de requisitos = definición detallada de lo que el sistema debe hacer

    Análisis = modelado de procesos y datos

    Diseño = creación de estructura técnica y lógica

    Codificación = desarrollo del software

    Pruebas = evaluación del sistema para detectar errores

    Implementación = instalación y entrega al usuario

    Mantenimiento = actualización y mejora del sistema

Si se pone una imagen de fondo en una página HTML, ¿por qué debe escogerse con cuidado?

  • Porque si no se hace, puede afectar de forma negativa a:

    La legibilidad del texto encima de la imagen.

    El rendimiento si el archivo es muy pesado.

    El cómo se adapta a distintos dispositivos.

    La coerencia entre el fondo y la página como tal.

Como ingeniero de software, ¿cuál es tu recomendación al elegir las unidades de un propiedad de estilo entre %, px y pt?

  • % (porcentaje):

    Se recomienda para diseños responsivos.

    Permite que los elementos se adapten al tamaño del contenedor.

    px (píxeles):

    Se recomienda para tamaños específicos y precisos (bordes, sombras, elementos pequeños).

    Es una unidad fija.

    pt (puntos):

    Está más orientado a impresión, no a pantallas.

    Por lo que usaría % para layouts flexibles, px para detalles específicos y evitaría pt en el desarrollo web moderno.

¿Por qué el uso de una versión minimizada del CSS mejora el rendimiento del sitio?

  • Un CSS minimizado elimina espacios innecesarios, los saltos de línea, comentarios y también reduce el tamaño del archivo.

    Esto provoca que el archivo pese menos, se descargue más rápido y mejore el tiempo de carga del sitio.

    Aunque el navegador interpreta igual el archivo, al ser más pequeño tarda menos en transferirse desde el servidor.
